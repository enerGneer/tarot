# 타로 카드 드로잉 사이트 — 요건정의서

## 1. 프로젝트 개요
- **목적**: 모바일 최적화된 3카드 타로 드로잉 도구. AI 해석용 프롬프트 텍스트를 클립보드에 복사하는 워크플로우 지원.
- **이용자**: 본인 + 동생 (2인 한정, 비공개 운용)
- **배포**: GitHub Pages (정적 호스팅)
- **참고 사이트**: [카니타로](https://canee.kr/tarot/pick.php?type=three-cards)

---

## 2. 기술 스택

| 항목 | 선택 | 비고 |
|------|------|------|
| 호스팅 | GitHub Pages | 서버리스, 정적 파일만 |
| HTML/JS | Vanilla JS (ES Modules) | 빌드 도구 없음, 단일 HTML + 모듈 분리 |
| 스타일 | SCSS → CSS | 모바일 퍼스트, 로컬 컴파일 후 CSS 커밋 |
| 상태 관리 | URL 파라미터 (query string) | 결과 페이지 영속성 확보 |
| 카드 이미지 | [RWS CC0 (itch.io)](https://luciellaes.itch.io/rider-waite-smith-tarot-cards-cc0) | CC0 라이선스, 웹용 사이즈 정리 완료 |

## 3. 카드 데이터

### 3.1 덱 구성
- **풀 덱 78장** (메이저 아르카나 22장 + 마이너 아르카나 56장)
- **역방향 없음** — 정방향만 취급

### 3.2 데이터 구조 (예시)
```json
{
    "id": 0,
    "name": "The Fool",
    "image": "00_the_fool.jpg"
}
```
- 카드명은 영문만 관리
- `suit`, `arcana` 등 분류 필드 불필요 (LLM이 카드명으로 판단 가능)
- 카드 데이터는 별도 JSON 파일(`cards.json`)로 분리하여 유지보수 용이하게 관리

## 4. 화면 구성 및 플로우

### 4.1 페이지 구조
단일 페이지(SPA), 2단계 화면 전환. URL 파라미터로 결과 상태 관리.

```
[드로잉 화면 (첫 화면)] → [결과 화면]
          ↑                      |
          └──── 다시 뽑기 ────────┘
```

### 4.2 드로잉 화면 (첫 화면)

페이지 진입 시 바로 표시. 질문 입력·배열 선택·카드 덱이 한 화면에 통합.
모든 입력 필드는 처음부터 노출되지만, 아무것도 선택/입력하지 않아도 카드 드로잉에 지장 없음.

#### 구성 요소 (상단 → 하단 순서)
1. **질문 입력** (input, 선택사항)
    - placeholder: `질문 :`
    - 처음부터 노출, 입력하지 않아도 무방
    - 입력 시 복사 텍스트에 포함됨
2. **배열(스프레드) 선택** (`<select>` 드롭다운)
    - 처음부터 노출, 기본값: 선택 없음
    - 프리셋은 `spreads.json`에서 로드 → JSON 파일 수정만으로 추가/삭제 가능
    - 커스텀 입력 기능 없음 (프리셋 전용)
    - 선택 시: 카드 덱 위에 위치명 슬롯 3개 표시 (예: `1. 과거` `2. 현재` `3. 미래`)
    - 미선택 시: 슬롯 표시 없음 또는 번호만 표시 (`1.` `2.` `3.`)
3. **카드 덱 영역** (겹침 배치)
    - 78장의 카드 뒷면을 겹쳐서 배치
    - **좌우 스크롤 없이 모바일 뷰포트(360px~) 내 완전 표시** (핵심 요건)
    - 레이아웃: 다행 × 겹침 배치 (행당 카드 수를 뷰포트 폭에 맞게 조절)
    - 카드 선택(탭) 시: 해당 카드가 위로 약간 올라가며 고정 (선택 애니메이션)
    - **한번 뽑은 카드는 되돌릴 수 없음** (잘못 뽑은 경우 처음부터 다시)
    - 선택 순서가 카드 순서 (1번째 탭 → 카드1, 2번째 → 카드2, 3번째 → 카드3)
4. **남은 카드 카운터 / 결과 보기 버튼**
    - 초기: "남은 카드 3"
    - 카드 선택 시마다 카운트 감소
    - 0이 되면: "선택 완료 — 결과 보기" 클릭 가능 버튼으로 전환
    - 클릭 시 URL 파라미터에 선택된 카드 ID를 기록하고 결과 화면으로 전환

#### 겹침 배치 레이아웃 상세
- **JS 동적 계산** 방식 채택 (컨테이너 폭 기준으로 행당 카드 수·행 수를 계산)
- 겹침 시 노출 폭(offset)은 고정값 유지 (터치 사용성 확보)
- `ResizeObserver`로 컨테이너 폭 변화 감지 → 반응형 자동 대응
- 계산 로직: `행당 카드 수 = Math.ceil(78 / 행 수)`, `겹침량 = cardWidth - offset`
- `margin-right: -(cardWidth - offset)` 로 겹침 적용

#### 카드 뒷면 디자인
- **CSS only** (배경색 + border)로 최소한으로 구현
- 별도 이미지 파일 없음, 추가 로딩 없음

### 4.3 결과 화면

#### 레이아웃 (상단 → 하단 순서)
1. **뒤로가기 버튼** (좌측 상단 배치)
    - 드로잉 화면(첫 화면)으로 복귀
2. **카드 3장 표시 영역**
    - 가로 배치, 모바일에서 좌우 스크롤 없이 뷰포트 내 완전 표시 (핵심 요건)
    - 카드 이미지 + 카드명 표시
    - 배열 선택 시: 각 카드 위에 위치명 라벨 표시
    - 배열 미선택 시: 위치명 없이 카드만 표시
3. **복사 텍스트 블록 + 복사하기 버튼**
    - 복사될 텍스트 내용을 블록으로 표시 (사용자가 내용 확인 가능)
    - 복사하기 버튼 클릭 시:
        - 클립보드 API (`navigator.clipboard.writeText`) 사용
        - 버튼 텍스트 "복사하기" → "복사완료"로 변경 (3초 후 원복)
        - 동시에 토스트 메시지 표시: "클립보드로 복사되었습니다"

#### 복사 텍스트 포맷

**질문 + 배열 모두 있을 때:**
```
질문 : {입력된 질문 텍스트}
스프레드 : {배열명}
1. {위치명1} : {카드1 영문명}
2. {위치명2} : {카드2 영문명}
3. {위치명3} : {카드3 영문명}
{프롬프트 보조 문구 — 추후 추가 예정}
```

**질문만 있을 때 (배열 미선택):**
```
질문 : {입력된 질문 텍스트}
1. {카드1 영문명}
2. {카드2 영문명}
3. {카드3 영문명}
{프롬프트 보조 문구 — 추후 추가 예정}
```

**질문 미입력 시 (배열 유무 불문):**
`질문 :` 행 자체를 생략. 카드 나열부터 시작.
```
1. {카드1 영문명}
2. {카드2 영문명}
3. {카드3 영문명}
{프롬프트 보조 문구 — 추후 추가 예정}
```

- 카드명은 영문만 사용
- 프롬프트 보조 문구: 별도 상수 또는 설정 파일에서 관리 (추후 커스텀 예정)

## 5. 핵심 요건 상세

### 5.1 모바일 레이아웃 — 좌우 스크롤 금지 (최우선 요건)
드로잉 화면과 결과 화면 모두 모바일 뷰포트(360px~430px)에서 좌우 스크롤이 발생하지 않아야 한다.

**드로잉 화면 (78장 겹침 배치)**:
- 카드 뒷면을 다행으로 나누어 겹침 배치
- 행당 카드 수 × 노출 폭(offset) + 카드 폭 ≤ 뷰포트 폭 - 좌우 패딩
- 구체적인 행 수/카드 배분은 뷰포트 폭 기반으로 동적 계산하거나, 고정 행 수(6행)로 설정
- 검증 기준: 360px 뷰포트에서 overflow-x 미발생

**결과 화면 (3장 가로 배치)**:
- `display: flex` + `gap` 활용
- 카드 너비: `calc((100vw - 좌우패딩 - gap합계) / 3)`
- 카드 이미지 비율 유지: `aspect-ratio: 2/3` 또는 `padding-top` 기법
- 최소 터치 영역 고려 (카드 폭 최소 90px 이상)

**공통 검증 기준**: Chrome DevTools 모바일 시뮬레이터 기준 iPhone SE(375px), iPhone 14(393px), Galaxy S series(360px~412px)에서 스크롤 미발생

### 5.2 URL 파라미터 기반 상태 관리
- **결과 화면 URL 형식**: `?q={질문}&cards={id1},{id2},{id3}&spread={spread_id}`
    - `q`: URL 인코딩된 질문 텍스트 (미입력 시 파라미터 생략)
    - `cards`: 카드 ID 3개 (쉼표 구분)
    - `spread`: 배열 프리셋 ID (미선택 시 파라미터 생략)
- **동작**:
    - 페이지 로드 시 URL 파라미터 존재 → 결과 화면 직접 렌더링
    - 뒤로가기 → 메인 화면 복귀 (브라우저 히스토리 활용)
    - 새로고침 → 동일 결과 유지

### 5.3 텍스트 복사 기능
- Clipboard API 사용 (`navigator.clipboard.writeText`)
- HTTPS 환경 필수 (GitHub Pages 기본 지원)
- 복사 성공 시 동시에 두 가지 피드백:
    1. 버튼 텍스트 "복사하기" → "복사완료"로 변경 (3초 후 원복)
    2. 토스트 메시지 "클립보드로 복사되었습니다" 표시 (자동 소멸)
- 복사 실패: fallback으로 `document.execCommand('copy')` 또는 에러 표시
